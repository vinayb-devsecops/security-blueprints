# Threat Modeling Workflow

Workflow ID: TM-001

## Objective

Identify, assess, and prioritize security threats during system design and architecture reviews.

## Inputs

- Architecture Diagrams
- Data Flows
- Trust Boundaries
- Business Requirements
- Security Requirements

## Methodology

### Step 1: Define Scope

- Application boundaries
- Components
- External integrations

### Step 2: Create Data Flow Diagram

Identify:

- Processes
- Data Stores
- Data Flows
- External Entities
- Trust Boundaries

### Step 3: Apply STRIDE

#### Spoofing
#### Tampering
#### Repudiation
#### Information Disclosure
#### Denial of Service
#### Elevation of Privilege

### Step 4: Risk Assessment

Evaluate:

- Likelihood
- Impact
- Existing Controls
- Residual Risk

### Step 5: Mitigation Planning

- Preventive Controls
- Detective Controls
- Corrective Controls

### Step 6: Tracking

- Open Risks
- Accepted Risks
- Remediation Items

## Outputs

- Threat Model
- Risk Register
- Security Requirements
- Mitigation Plan
